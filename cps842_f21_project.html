CPS842: Information Retrieval and Web Search

Fall 2021

Project (Due: Dec. 12, 2021)

Project Ideas:
You can choose the project idea from the following list, or propose your own as long as it is related to the course topic and the workload is adequate. In the latter case, you need to email me a short proposal by Nov. 18, 2021.

1. Build your own search engine. You can develop your own crawler or use an existing crawler such as Nutch to crawl the web. You can decide the starting seed set, which could be a set of pages you are interested in, or pages from web directories. You can use a HTML parser to parse the HTML pages and then use your indexing program from assignment 1 to build the inverted index. You may need to modify the program because the web collection is different from the CACM collection. Your search engine should have a web interface to accept keyword-based queries. The results returned to the user should be ranked. You can use the vector space model to implement the ranking algorithm. You should also implement a link-based ranking algorithm such as PageRank. The final ranking score would be the combination of the two, for example, score(d, q) = w1*cos-score(d, q) + w2*pagerank(d) where w1+w2=1. You can implement the PageRank algorithm by yourself or use an existing implementation (e.g., Jung). In the latter case, you need to cite the source in your report.

Note: You can restrict your search engine to only support HTML pages. Also, you can focus on a particular domain. You should download minimum 500 web pages. The crawler program should be separate so that you don't need to run it during the demo.

2. Implement a movie recommender system. Set up a web site in which a user can rate a movie he/she has watched. You can import the movie titles and other related information from IMDB. The rating is a number between 0 and 5. After you collect a certain amount of ratings from users, you can run the recommender system, which will then base on the previous ratings given by the user, recommend new movies to the user. You can also import an existing rating dataset (e.g. MovieLens), in this case, a web interface for users to rate a movie is still required. You can implement one of the recommendation algorithms we have learned in class (e.g., content-based, user/item-based collaborative filtering algorithm). You are also welcome to compare your model with other popular recommendation models (e.g., Matrix Factorization, Deep Neural Network). For other models, you can use an existing library. For comparison, you can report the RMSE and precision/recall values of different algorithms when running on a certain dataset.

Note: You have to implement at least one recommendation algorithm by yourself (without using an existing library). Your program should have a basic user management function, e.g., for a user to register, log in, etc.

3. Implement a document clustering system. The dataset can be downloaded here: http://mlg.ucd.ie/datasets/bbc.html. There are two datasets, one is BBC news and the other is BBC Sports news. You can choose either one, please download the raw text files. There are 5 classes of articles in each dataset. These class labels can be used as the gold standard when you measure the external evaluation metrics. You can use the programs from the two assignments (with some modifications) to process the documents and build the TFIDF weight vectors for these documents. Then, you can implement one of the clustering algorithms and run it on this dataset. Two options are: k-means clustering and hierachical agglomerateive clustering (with a selected similarity threshold, you can also get k clusters). Calculate the following metrics: tightness of each cluster (sum of squared distances between each document in the cluster and the centroid of the cluster), one of the external criteria (purity or rand index).

Note: Interface is required. The input required from the user is the k value. After you run the program, the output should be the clustered results (e.g., similar to the Scatter/Gather example we show in class). For each cluster, show the cluster ID, 5 terms with the highest TFIDF scores (the combined scores from all the documents in the cluster), and all the documents (ID + title) in the cluster. Please also show the actual class label beside each document. At the end, you need to show the values of the chosen metrics.
